# 踩坑记录（Convert_to_MMD5）

把开发/测试 XPS→MMD 转换时反复踩到的坑集中记一处，避免下次重蹈。
每条给「现象 → 原因 → 正确做法」。新坑往对应分组追加。

来源汇总：CLAUDE.md、`docs/TESTING.md`、`docs/skirt_physics_design.md` 及历次会话记忆。

---

## 一、测试 & 尺度（最容易崩的地方）

### 1. 比对必须先缩放到目标等高
- **现象**：转换出来的「米」尺度小模型直接和目标 PMX 并排，位移/偏差只有几毫米，肉眼看不出对错。
- **原因**：转换模型是米尺度，目标 PMX 约 12× 大（本测试集缩放比 **11.91×** = 目标身高 / 转换身高）。
- **做法**：验证前用根 empty 的 scale 把转换模型放大到与目标等高再比。

### 2. VMD 导入尺度搞错 → IK 腿崩 60~100°
- **现象**：给转换模型导入 VMD 后，腿部 IK 角度炸到 60~100°（FK 旋转正常，不受影响）。
- **原因**：转换模型是米尺度，VMD 的平移分量是 PMX 尺度。尺度不匹配，IK 解算把腿拉崩。
- **做法**：
  - 转换模型导入 VMD：`scale = 1 / 缩放比 ≈ 0.084`。
  - 目标 PMX 导入 VMD：`scale = 1.0`。
  - FK 旋转与尺度无关，所以「只有 IK 崩、FK 正常」就是这个尺度坑的典型信号。

### 3. VMD 必须从干净场景一次性按正确 scale 导入
- **现象**：足 IK 首帧被污染，对比数据莫名其妙。
- **原因**：先用 scale=1.0 导一遍、清空、再用正确 scale 重导，残留污染了足 IK 首帧。
- **做法**：干净场景里**一次性**按正确 scale 导入，不要先错后清重导。

### 4. 骨名映射两边取法不同
- **现象**：脚本里按同一字段取骨名，一边空。
- **原因**：转换模型 MMD 名在 `bone.name`（如 `左腕`）；目标 PMX 在 `mmd_bone.name_j`。
- **做法**：对比时分别取，别用同一字段。

### 5. FK σ≈0.5~0.8 的小抖动不算真失败
- **现象**：走了 A-Pose 工具后端到端自检 FK σ 不是 0，疑似 FAIL。
- **原因**：MMD6 移植的朴素 A-pose 法保留源左右不对称，固有 σ≈0.5~0.8（canonical 方案才是 σ=0）。
- **做法**：σ≈0.5~0.8 视觉/动作无碍，**不算真失败**；真正看腿 IK 最差帧 + 渲染图。判 PASS/FAIL 以腿 IK 为准。

### 6. 14、15 是两步，别只点一步
- **现象**：付与变换没生效。
- **原因**：`setup_mmd_grants`（步14）只设标志，**不 apply**；必须再点步15 `mmd_tools.apply_additional_transform`（应用付与变换）。
- **做法**：手动分步时 14+15 连着点。

## 二、本机环境 & 装插件（macOS Blender 3.6）

### 7. 必须确认生效引擎，否则跑的是旧版本
- **现象**：改了仓库代码不生效，行为像旧插件。
- **原因**：同类旧插件（如 `Convert_to_MMD4`）没禁用，或 `sys.modules` 里残留旧模块。
- **做法**：热重载顺序 = `addon_utils.disable` 旧插件 → 清掉 `sys.modules` 里所有 `Convert_to_MMD5*` → `addon_utils.enable("Convert_to_MMD5")` → `save_userpref()`。
  改完务必核对：`bpy.types.OBJECT_OT_one_click_convert.__module__ == "Convert_to_MMD5.convert.pipeline"`。

### 8. XNALaraMesh 模块名带 `-master`
- **现象**：按 `XNALaraMesh` 找模块/启用失败。
- **原因**：本机依赖目录名是 `XNALaraMesh-master`。
- **做法**：引用模块名带 `-master`。

### 9. 端到端脚本要跳过 `__main__` 自动跑
- **现象**：`exec` 脚本时它自己就跑起来了，没法用自定义参数。
- **原因**：脚本底部有 `__main__` 自动执行块。
- **做法**：用 `g={"__name__":"c2m_test"}` 作命名空间 exec，再手动调 `g["run"]()`。

### 10. 想跳过某流水线步，patch 别改源码
- **做法**：临时 patch `Convert_to_MMD5.convert.pipeline.PIPELINE_PRE_D/POST_D`（过滤掉某 `step[0]`），跑完在 `finally` 还原。不动源码。

## 三、macOS Downloads 权限坑（TCC）

### 11. Bash/Read 工具读不了 `~/Downloads`，但 Blender 能
- **现象**：用 Bash/Read 读 `~/Downloads/...` 报 `Operation not permitted`。
- **原因**：macOS TCC 给 Claude 的 Bash/Read 工具没授权 Downloads；Blender 进程本身有权限。
- **做法**：
  - 核对素材用 Blender 的 `os.listdir` / `open`。
  - 渲染图：Blender 先写到 Downloads，再用 Blender 的 `shutil.copy2` 拷到 session scratchpad，然后才能 Read 查看。

## 四、A-Pose 工具（MMD6 移植，手动可选步骤 0.5）

### 12. 源 T-Pose 才转，A-pose 跳过；不强求贴目标手臂
- **判定**：自检里「手臂向下角」≈0~2°（手臂接近水平）= T-Pose，先点「0.5 转换为 A-Pose」再走流程；源已是 A-pose 则跳过。
- **理由**：转 A-pose 是为了让转换模型本身符合 MMD 规范（形变更干净），**不是去贴某个具体目标 PMX**。即使并排目标恰好也是 T-pose（手臂对不上），也照转不误。

### 13. 已知瑕疵（用户接受「就这样留着」）
- `target_angle=36` 不是真下垂角，实测手臂落到 ~54°（目标 ~37°，溢出）。
- 保留源左右不对称 → FK σ≈0.5~1.2（见坑 #5）。
- 烘焙时复制的骨架修改器在栈底 apply，报 `modifier was not first`（功能不受影响）。
- 要 σ=0 的更干净结果得走 canonical 方案，当前不做。

## 五、衣服权重 & 裙子物理

### 14. 衣服权重转移的选择顺序
- **约定**：**先选衣服，最后加选身体**（身体 = 活动来源 active）。
- **原因**：Data Transfer（VGROUP_WEIGHTS, Nearest Face Interpolated）从 active 对象取源权重。顺序反了就把衣服权重转给身体。
- **做法**：`object.transfer_clothing_weights` 跑完会归一 + Limit Total≤4 + 补 Armature 修改器。

### 15. 裙子物理：尺度无关参数抄目标，尺寸必须按转换模型重算
- **现象**：直接抄目标 PMX 的刚体 BOX size，裙子刚体大得离谱。
- **原因**：目标尺寸是它自己 ≈12× 的 PMX 尺度；转换模型是米尺度。
- **做法**：
  - **尺度无关**（mass/damping/friction/group/碰撞掩码/角限位/弹簧）直接抄目标。
  - **尺寸**（刚体 BOX 长/宽/厚）按转换模型骨骼几何重算：长=骨段长、宽=相邻片间距/2、厚=0.15×长。

### 16. 裙根挂在 `unused trash` 也要锚到 `下半身`
- **现象**：某些模型（如 cartilla）裙根骨父级是 `unused trash 17`，不是 `下半身`。
- **做法**：裙链 seg1 的关节统一连到 `下半身` kinematic 锚（没有就建），**不管裙根当前父级是谁**。

## 六、远端部署（BlenderMCP，见 TESTING.md）

### 17. 打包要禁 macOS 资源叉
- **做法**：`COPYFILE_DISABLE=1 tar czf ...`，否则 tar 里混进 `._*` 资源叉文件。

### 18. 面板上传会把 basename 追加到目录后
- **现象**：上传 `cmmd2.tgz` 到 `<REMOTE_TMP>/cmmd2.tgz`，实际落点是 `<REMOTE_TMP>/cmmd2.tgz/cmmd2.tgz`。
- **做法**：解压时按真实落点取路径。

### 19. 9876 拒绝连接 = 隧道挂了
- **做法**：每次先 `mcp__blender__test_connection` 期望 `Connection OK!`；拒连按 `remote.local.md` 重启隧道。机器相关真实值（主机/密码/路径/素材）全在 gitignore 的 `docs/remote.local.md`。
